# 🧾 SQL Sales Insights Project – Business Analytics & KPIs

## 🔗 Project Foundation – Data Warehouse

This project focuses on **analyzing structured sales data and generating actionable business insights using SQL**.

👉 The dataset is built using a data warehouse project:  
➡️ [SQL Data Warehouse Project](https://github.com/Sreya200402/SQL_Data_Warehouse_Project)

This repository represents the **analytics layer**, where raw data is transformed into meaningful insights.

---

## 🎯 Business Problem

Even with a well-designed data warehouse, businesses often struggle to:

- Track key performance metrics  
- Identify revenue-driving customers and products  
- Understand sales trends over time  
- Detect underperforming areas  

👉 This project solves these challenges using **SQL-driven analytics**.

---

## 📊 Project Workflow

This project follows a structured analytics approach:

### 🔹 1. Exploratory Data Analysis (EDA)
- Database exploration  
- Dimension & date analysis  
- KPI identification (Revenue, Orders, Customers)  

### 🔹 2. Core Analysis
- **Magnitude Analysis** → Sales distribution across products  
- **Ranking Analysis** → Top/Bottom customers & products  
- **Trend Analysis** → Sales over time  
- **Cumulative Analysis** → Running totals & growth patterns  

### 🔹 3. Advanced Analytics
- **Performance Analysis** → Product & customer performance  
- **Segmentation** → Grouping customers/products  
- **Part-to-Whole Analysis** → Contribution to total revenue  

### 🔹 4. Reporting
- Customer-level insights  
- Product-level insights  

---

## ⚙️ Tech Stack

- SQL Server  
- SQL (Advanced Queries)  
- Data Warehouse (Gold Layer)  

---

## 🧠 Key SQL Concepts Used

- Aggregations (`SUM`, `COUNT`, `AVG`)  
- Joins (Fact & Dimension tables)  
- Window Functions (`ROW_NUMBER`, `RANK`, `SUM OVER`)  
- CTEs (Common Table Expressions)  
- Time-based analysis  

---

## 💡 Key Insights

- A small percentage of customers contributes to a large portion of revenue  
- Sales trends show clear seasonality across months  
- Certain product categories consistently outperform others  
- Some segments generate low revenue despite high volume

## 📁 Project Structure

The repository is organized into modular SQL scripts:

- `01–04` → Data Exploration & KPI Setup
